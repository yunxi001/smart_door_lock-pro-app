# 文档维护指南

本指南说明如何维护智能门锁 Pro 项目的文档体系。

## 📁 文档体系结构

### 三层文档架构

```
智能门锁-pro/
├── README.md                    # 第一层：项目入口文档
├── docs/                        # 第二层：详细技术文档
│   ├── README.md                # 文档索引
│   ├── 文档导航.md               # 快速查找指南
│   ├── 文档维护指南.md           # 本文件
│   ├── DOCUMENTATION.md         # 完整技术文档
│   ├── *.md                     # 各类专项文档
│   └── archive/                 # 归档文档
└── .kiro/steering/              # 第三层：AI 辅助开发规范
    ├── product.md               # 产品功能说明
    ├── structure.md             # 项目结构规范
    ├── tech.md                  # 技术栈规范
    └── capacitor.md             # Capacitor 规范
```

### 文档分类与职责

| 文档类型     | 存放位置           | 更新频率         | 目标读者     |
| ------------ | ------------------ | ---------------- | ------------ |
| **入口文档** | 根目录 `README.md` | 每次重大功能更新 | 所有人       |
| **技术文档** | `docs/` 目录       | 功能变更时更新   | 开发者、测试 |
| **协议文档** | `docs/` 目录       | 协议版本升级时   | 前后端开发者 |
| **构建文档** | `docs/` 目录       | 构建流程变更时   | 移动端开发者 |
| **开发规范** | `.kiro/steering/`  | 架构调整时更新   | AI 辅助开发  |
| **归档文档** | `docs/archive/`    | 不再更新         | 参考用途     |

## 📝 文档更新规范

### 何时更新文档

| 场景         | 需要更新的文档                  | 优先级 |
| ------------ | ------------------------------- | ------ |
| 新增核心功能 | `README.md`, `DOCUMENTATION.md` | 🔴 高  |
| 修改现有功能 | `DOCUMENTATION.md`              | 🟡 中  |
| 协议版本升级 | 协议文档，归档旧版本            | 🔴 高  |
| 修复 Bug     | 可选更新 `DOCUMENTATION.md`     | 🟢 低  |
| 构建流程变更 | 构建相关文档                    | 🟡 中  |
| 架构调整     | `.kiro/steering/` 规范文档      | 🔴 高  |
| 任务完成     | 归档相关文档                    | 🟡 中  |

### 文档更新检查清单

#### ✅ 新增功能时

- [ ] 更新 `README.md` 核心功能列表
- [ ] 在 `DOCUMENTATION.md` 添加功能模块说明
- [ ] 更新 `docs/README.md` 索引（如需要）
- [ ] 更新 `.kiro/steering/product.md`（如涉及产品功能）
- [ ] 更新 `.kiro/steering/structure.md`（如涉及架构变更）

#### ✅ 协议升级时

- [ ] 创建新版本协议文档（如 `v2.6.md`）
- [ ] 归档旧版本到 `docs/archive/old-versions/`
- [ ] 更新 `docs/README.md` 中的协议文档链接
- [ ] 更新 `DOCUMENTATION.md` 中的协议版本号
- [ ] 创建升级说明文档（如需要）

#### ✅ 任务完成时

- [ ] 将任务相关文档移至 `docs/archive/`
- [ ] 更新 `docs/archive/README.md` 归档索引
- [ ] 从 `docs/README.md` 移除，添加归档链接
- [ ] 更新归档时间记录

## 🔄 文档归档流程

### 归档时机

**应该归档的文档：**

- ✅ 已完成任务的实施文档（如迁移报告、优化报告）
- ✅ 历史版本的协议文档
- ✅ 已解决的问题排查记录
- ✅ 过时的设计文档
- ✅ 项目阶段性总结（如毕业设计总结）

**不应归档的文档：**

- ❌ 当前使用的协议文档
- ❌ 构建与部署指南（除非流程已废弃）
- ❌ 核心技术文档 `DOCUMENTATION.md`
- ❌ 开发规范文档 `.kiro/steering/`

### 归档步骤

1. **确认归档**

   ```bash
   # 确认文档不再需要频繁更新
   # 确认文档内容已过时或任务已完成
   ```

2. **移动文件**

   ```bash
   # 将文档移至对应的归档子目录
   mv docs/某文档.md docs/archive/对应分类/
   ```

3. **更新索引**
   - 在 `docs/archive/README.md` 添加归档记录
   - 更新归档时间和分类说明

4. **更新主文档**
   - 从 `docs/README.md` 移除该文档
   - 添加指向归档位置的链接（如需要）

5. **提交变更**
   ```bash
   git add docs/
   git commit -m "docs: 归档某文档"
   ```

### 归档目录规范

```
docs/archive/
├── README.md                          # 归档索引
├── 项目总结文档.md                     # 项目总结类
├── protocol-upgrade-vX.X/             # 协议升级记录
├── color-system/                      # 设计系统
├── accessibility/                     # 测试报告
├── optimization-reports/              # 优化记录
├── old-versions/                      # 历史版本
└── troubleshooting/                   # 问题排查
```

## ✍️ 文档编写规范

### Markdown 格式规范

#### 标题层级

```markdown
# 一级标题 - 文档标题

## 二级标题 - 主要章节

### 三级标题 - 子章节

#### 四级标题 - 详细说明
```

#### 表格使用

```markdown
| 列1  | 列2  | 列3  |
| ---- | ---- | ---- |
| 内容 | 内容 | 内容 |
```

#### 代码块

````markdown
```typescript
// TypeScript 代码
const example = "示例";
```

```bash
# Shell 命令
npm install
```
````

#### 链接规范

```markdown
# 相对链接（推荐）

[文档名称](./相对路径.md)

# 绝对链接（外部资源）

[外部资源](https://example.com)
```

### 内容编写规范

#### 1. 文档头部信息

```markdown
# 文档标题

简短的文档说明（1-2句话）

**最后更新：** YYYY-MM-DD  
**维护者：** 团队名称
```

#### 2. 目录结构（长文档）

```markdown
## 目录

- [章节1](#章节1)
- [章节2](#章节2)
  - [子章节2.1](#子章节21)
```

#### 3. 代码示例

- 提供完整的可运行代码
- 添加必要的注释
- 说明前置条件和预期结果

#### 4. 表格使用

- 用于对比、配置说明、参数列表
- 保持列宽适中，避免过长内容

#### 5. 图标使用

```markdown
✅ 正确做法
❌ 错误做法
🔴 高优先级
🟡 中优先级
🟢 低优先级
📚 文档
🔍 查找
⚠️ 警告
```
