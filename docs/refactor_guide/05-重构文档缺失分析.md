### 1. 命令响应匹配机制未定义

**问题描述：**
文档删除了 `seq_id`，但没有明确说明 **App如何匹配命令和响应**。

**当前文档说法：**

> "ack 的匹配通过 type 字段和时序判断，或简化为'下一个ack就是当前命令的响应'"

**问题分析：**

1. **并发命令场景未考虑**：用户快速点击多个按钮（如先开锁，再开灯），如何区分哪个ack对应哪个命令？
2. **"时序判断"不具体**：是FIFO队列？还是只允许单命令？
3. **"下一个ack"过于简化**：如果中间收到 `status_report` 或 `event_report`，如何处理？

**补充建议：**

#### 方案A：单命令模式（推荐，最简单）

```typescript
// App端实现：同一时间只允许一个命令在执行
class DeviceService {
  private commandInProgress: boolean = false;
  private pendingResolve: ((value: any) => void) | null = null;
  private pendingReject: ((error: any) => void) | null = null;

  async sendCommand(command: object): Promise<void> {
    // 如果有命令正在执行，拒绝新命令
    if (this.commandInProgress) {
      throw new Error("请等待上一个命令完成");
    }

    return new Promise((resolve, reject) => {
      this.commandInProgress = true;
      this.pendingResolve = resolve;
      this.pendingReject = reject;

      // 发送命令
      this.ws.send(JSON.stringify(command));

      // 10秒超时
      setTimeout(() => {
        if (this.commandInProgress) {
          this.commandInProgress = false;
          reject(new Error("命令超时，请重试"));
        }
      }, 10000);
    });
  }

  private handleAck(msg: any) {
    if (!this.commandInProgress) {
      // 收到未预期的ack，忽略
      return;
    }

    this.commandInProgress = false;

    if (msg.code === 0) {
      this.pendingResolve?.();
    } else {
      this.pendingReject?.(new Error(msg.msg || "命令执行失败"));
    }
  }
}
```

**优点：**

- 实现简单，逻辑清晰
- 不会出现响应混乱
- 符合"简化"的设计原则

**缺点：**

- 用户体验略差（需要等待）
- 但可以通过UI禁用按钮来优化

````
**补充建议：**

```markdown
### 监控模式二进制流规范

**协议保持不变：** 继续使用 BinaryProtocol2（16字节头部）

**帧格式：**
````

+----------------+----------------+--------------------------------+
| version(2B) | type(2B) | reserved(4B) |
+----------------+----------------+--------------------------------+
| timestamp(4B) | payload_size(4B) |
+----------------+----------------+--------------------------------+
| payload data... |
+------------------------------------------------------------------+

````

**type 字段：**
- `0` + reserved=0: 音频流（OPUS编码，16kHz）
- `0` + reserved=(width<<16)|height: 视频流（JPEG编码）

**App端处理：**
```typescript
private handleBinaryMessage(data: ArrayBuffer) {
  const view = new DataView(data);
  const version = view.getUint16(0);
  const type = view.getUint16(2);
  const reserved = view.getUint32(4);
  const timestamp = view.getUint32(8);
  const payloadSize = view.getUint32(12);
  const payload = data.slice(16);

  if (type === 0 && reserved === 0) {
    // 音频帧
    this.handleAudioFrame(payload);
  } else if (type === 0 && reserved !== 0) {
    // 视频帧
    const width = (reserved >> 16) & 0xFFFF;
    const height = reserved & 0xFFFF;
    this.handleVideoFrame(payload, width, height);
  }
}
````

**注意：** 此部分协议完全保留，无需修改。

````markdown
```typescript
// hooks/useCommandDebounce.ts
import { useState, useCallback } from "react";

export function useCommandDebounce(
  command: () => Promise<void>,
  debounceMs: number = 2000,
) {
  const [isExecuting, setIsExecuting] = useState(false);

  const execute = useCallback(async () => {
    if (isExecuting) {
      // 可选：显示提示"请等待上一个操作完成"
      return;
    }

    setIsExecuting(true);
    try {
      await command();
    } catch (error) {
      // 错误处理
      console.error(error);
    } finally {
      setTimeout(() => {
        setIsExecuting(false);
      }, debounceMs);
    }
  }, [command, debounceMs, isExecuting]);

  return { execute, isExecuting };
}
```
````

```typescript
class DeviceService {
  private reconnectAttempts = 0;
  private maxReconnectAttempts = 5;
  private reconnectDelay = 2000; // 2秒

  private handleDisconnect() {
    if (this.reconnectAttempts < this.maxReconnectAttempts) {
      this.reconnectAttempts++;
      this.emit("log", {
        msg: `连接断开，${this.reconnectDelay / 1000}秒后重连 (${this.reconnectAttempts}/${this.maxReconnectAttempts})`,

      setTimeout(() => {
        this.connect(this.lastUrl, this.lastDeviceId, this.lastAppId);
      }, this.reconnectDelay);

      // 指数退避
      this.reconnectDelay = Math.min(this.reconnectDelay * 2, 30000);

      this.emit("log", {

      });
    }
  }
```

```typescript
// 使用 WeakMap 避免内存泄漏
class DeviceService {
  private listenersByEvent = new Map<string, Set<EventCallback>>();

  public on(eventName: string, callback: EventCallback) {
    if (!this.listenersByEvent.has(eventName)) {
      this.listenersByEvent.set(eventName, new Set());
    }
    this.listenersByEvent.get(eventName)!.add(callback);

    return () => {
      this.listenersByEvent.get(eventName)?.delete(callback);
    };
  }

  private emit(type: string, data: any) {
    this.listenersByEvent.get(type)?.forEach((cb) => cb(type, data));
  }
}
```
