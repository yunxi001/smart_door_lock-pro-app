## 📝 文档维护规范

### 文档分类原则

| 类型         | 存放位置          | 说明                                         |
| ------------ | ----------------- | -------------------------------------------- |
| **当前文档** | `docs/` 根目录    | 保持最新版本，定期更新，对项目开发有指导意义 |
| **归档文档** | `docs/archive/`   | 已完成任务、历史版本、过时文档               |
| **开发规范** | `.kiro/steering/` | AI 辅助开发的项目规范（产品、架构、技术栈）  |

### 命名规范

- 使用中文，清晰描述文档内容
- 协议文档使用版本号后缀（v2.x）
- 归档文档按类型分目录存放

### 归档时机

- 功能完成：任务完成后将相关文档归档
- 版本迭代：新版本发布后将旧版本文档归档
- 文档过时：内容不再适用时及时归档
