## 📝 归档规范

### 何时归档

| 场景             | 说明                                   | 示例                 |
| ---------------- | -------------------------------------- | -------------------- |
| **任务完成**     | 功能开发完成，相关文档不再需要频繁更新 | 色彩系统迁移完成     |
| **版本迭代**     | 新版本发布，旧版本文档归档             | v2.4 → v2.5 协议升级 |
| **文档过时**     | 内容不再适用于当前项目                 | 旧的构建配置         |
| **项目阶段完成** | 项目里程碑达成                         | 毕业设计完成         |

### 归档流程

1. **确认归档**: 确认文档不再需要频繁更新
2. **分类存放**: 按类型放入对应的归档子目录
3. **更新索引**: 在本 README 中添加归档记录
4. **更新主文档**: 从 `docs/README.md` 中移除，添加归档链接
