# HomeScreen 视觉验证指南

本文档提供首页配色迁移的手动视觉验证步骤。

## 测试环境准备

1. 启动开发服务器：`npm run dev`
2. 在浏览器中打开应用
3. 准备测试浅色和深色两种模式

## 验证检查清单

### 1. 设备状态卡片

#### 浅色模式

- [ ] 卡片背景为白色，边框为浅灰色
- [ ] 设备在线：绿色圆点 + 绿色文字
- [ ] 设备离线：红色圆点 + 红色文字
- [ ] 连接中：橙色圆点（脉动动画）+ 橙色文字
- [ ] 电量显示颜色正确（低电量红色，中等黄色，充足绿色）

#### 深色模式

- [ ] 卡片背景为深灰色，边框为中灰色
- [ ] 状态指示器颜色适配深色模式（饱和度降低）
- [ ] 文本清晰可读，对比度充足

### 2. 核心开锁按钮

#### 浅色模式

- [ ] 锁定状态：蓝色背景 + 蓝色图标
- [ ] 开启状态：绿色背景 + 绿色图标
- [ ] 禁用状态：灰色背景 + 灰色图标
- [ ] 悬停时背景色加深
- [ ] 点击时有缩放动画

#### 深色模式

- [ ] 按钮背景适配深色模式
- [ ] 图标颜色清晰可见
- [ ] 悬停效果明显

### 3. 快捷功能区

#### 浅色模式

- [ ] 临时密码：橙色背景 + 橙色图标
- [ ] 补光灯：灰色背景（关闭）/ 黄色背景（开启）
- [ ] 门铃测试：青色背景 + 青色图标
- [ ] 禁用状态为灰色

#### 深色模式

- [ ] 所有按钮适配深色背景
- [ ] 图标和文字清晰可读
- [ ] 悬停效果明显

### 4. 最近动态列表

#### 浅色模式

- [ ] 开门成功记录：浅绿色背景 + 绿色图标
- [ ] 拒绝访问记录：浅红色背景 + 红色图标
- [ ] 其他事件：浅灰色背景 + 青色图标
- [ ] 时间戳为灰色小字
- [ ] 悬停时背景色加深

#### 深色模式

- [ ] 记录背景适配深色模式
- [ ] 文本清晰可读
- [ ] 悬停效果明显

### 5. 临时密码弹窗

#### 浅色模式

- [ ] 遮罩层为半透明深灰色，带模糊效果
- [ ] 弹窗背景为白色
- [ ] 密码显示区域为浅灰色背景 + 蓝色文字
- [ ] 取消按钮为灰色边框
- [ ] 确认按钮为蓝色背景 + 白色文字

#### 深色模式

- [ ] 遮罩层更深
- [ ] 弹窗背景为深灰色
- [ ] 密码显示区域适配深色模式
- [ ] 按钮颜色清晰可见

## 对比度验证

使用浏览器开发者工具或在线对比度检查工具验证：

- [ ] 所有正常文本对比度 ≥ 4.5:1
- [ ] 所有大文本对比度 ≥ 3:1
- [ ] 所有交互元素对比度 ≥ 3:1
- [ ] 深色模式下对比度同样符合标准

## 动画和交互

- [ ] 连接中状态的脉动动画流畅
- [ ] 按钮悬停时颜色过渡平滑
- [ ] 按钮点击时缩放效果自然
- [ ] 主题切换时颜色过渡流畅

## 响应式测试

- [ ] 在不同屏幕尺寸下布局正常
- [ ] 移动端触摸交互正常
- [ ] 所有元素在小屏幕上可见且可操作
